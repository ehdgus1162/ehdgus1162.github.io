## 1. 변수 선언과 사용

변수(variable)는 값을 저장하기 위한 메모리 공간에 부여된 이름입니다. 프로그램은 변수를 통해 메모리에 값을 저장하고 읽을 수 있습니다.

---

### 1.1 변수의 개념과 특징

- 변수는 **데이터를 저장하는 메모리 공간의 이름**입니다.
- 자바에서 변수는 선언된 타입의 값만 저장할 수 있습니다(강한 타입 언어).
- 변수를 통해 값을 효율적으로 관리하고 재사용할 수 있습니다.

> **Java 10 이후 타입 추론**: 
> Java 10부터는 지역 변수에 한해 타입 추론(type inference)이 지원됩니다.
> `var` 키워드를 사용하면 컴파일러가 초기값을 기준으로 변수의 타입을 자동으로 결정합니다.
> ```java
> var number = 10;    // int로 추론됨
> var message = "Hello";  // String으로 추론됨
> ```

### 1.2 변수 선언과 초기화 과정

변수 사용은 크게 **선언**과 **초기화** 두 단계로 이루어집니다:

1. **변수 선언**: 타입과 이름을 지정하여 변수를 정의합니다.
   ```java
   int score;   // int 타입의 score 변수 선언
   ```

2. **변수 초기화**: 선언된 변수에 최초로 값을 할당합니다.
   ```java
   score = 90;  // score 변수 초기화
   ```

3. **선언과 초기화 동시에**: 한 문장으로 선언과 초기화를 함께 할 수 있습니다.
   ```java
   int score = 90;  // 선언과 동시에 초기화
   ```

**중요**: 변수는 초기화된 후에만 사용할 수 있습니다. 초기화되지 않은 변수를 사용하면 컴파일 오류가 발생합니다.

```java
int value;             // 변수 선언만 함
int result = value + 10;  // 오류: 초기화되지 않은 변수 사용
```

### 1.3 변수 사용과 값 교환

변수는 다른 변수에 대입하여 값을 복사할 수 있습니다. 이를 통해 다양한 데이터 조작이 가능합니다.

**변수 값 교환 예제**:
```java
// 두 변수의 값 교환하기
int x = 10;
int y = 5;

// 임시 변수를 사용한 교환 방법
int temp = x;  // x의 값을 임시 저장
x = y;         // x에 y의 값 대입
y = temp;      // y에 임시 저장된 x의 원래 값 대입

System.out.println("x: " + x);  // 출력: x: 5
System.out.println("y: " + y);  // 출력: y: 10
```

## 2. 기본 타입(Primitive Types)

자바는 8개의 기본 타입(primitive type)을 제공합니다. 기본 타입 변수는 실제 값을 직접 저장합니다.

### 2.1 기본 타입 분류

자바의 기본 타입은 크게 세 가지 범주로 분류됩니다:

| 분류 | 타입 | 용도 |
|------|------|------|
| **정수 타입** | `byte`, `short`, `int`, `long`, `char` | 정수 값 저장 |
| **실수 타입** | `float`, `double` | 실수 값 저장 |
| **논리 타입** | `boolean` | true/false 값 저장 |

### 2.2 정수 타입의 특징과 범위

정수 타입은 크기와 표현 범위에 따라 다양한 옵션을 제공합니다:

| 타입 | 크기 | 값 범위 | 주요 용도 |
|------|------|---------|----------|
| `byte` | 1바이트 | -128 ~ 127 | 파일 또는 네트워크 전송 시 바이트 단위 데이터 처리 |
| `short` | 2바이트 | -32,768 ~ 32,767 | C/C++ 언어와의 호환성을 위해 사용 |
| `int` | 4바이트 | -2,147,483,648 ~ 2,147,483,647 | 정수 연산의 기본 타입 |
| `long` | 8바이트 | -9,223,372,036,854,775,808 ~ 9,223,372,036,854,775,807 | int 범위를 넘어서는 큰 정수 값 저장 |

**바이트와 비트의 관계**:
- 1바이트(byte) = 8비트(bit)
- 비트(bit)는 0과 1을 저장하는 최소 단위
- n비트로 표현할 수 있는 값의 범위: -2<sup>n-1</sup> ~ 2<sup>n-1</sup>-1

**정수 리터럴 표기법**:
```java
int decimalValue = 10;       // 10진수
int octalValue = 012;        // 8진수 (0으로 시작)
int hexValue = 0xA;          // 16진수 (0x로 시작, A,B,C,D,E,F로 작성)
int binaryValue = 0b1010;    // 2진수 (0b로 시작, Java 7부터)

// long 타입 리터럴은 끝에 L 또는 l을 붙임
long bigNumber = 9223372036854775807L;
```

> 기본적으로 컴파일러는 정수 리터럴을 int 타입 값을 간주하기 때문에
> int 타입의 허용 범위를 초과하는 리터럴은 뒤에 'l', 'L'을 붙여 long 타입 값임을 지정해야 합니다.

### 2.3 문자 타입 (char)

자바에서 단일 문자를 표현할 때는 `char` 타입을 사용합니다. 문자 리터럴은 작은따옴표(`'`)로 감싸 표현합니다.

`char` 타입은 내부적으로 **2바이트(16비트)** 크기의 메모리를 사용하며, **0부터 65535까지의 유니코드 값**을 저장할 수 있습니다. 유니코드는 전 세계의 모든 문자를 고유한 숫자에 매핑한 국제 표준 규약으로, 자바는 이 유니코드를 기반으로 문자를 처리합니다.

```java
char var1 = 'A';     // 'A' 문자는 내부적으로 유니코드 값 65로 저장됨
char var2 = 65;      // 숫자 65를 직접 대입해도 'A'가 저장됨
char var3 = '가';    // '가' 문자는 내부적으로 유니코드 값 44032로 저장됨
char var4 = 44032;   // 숫자 44032를 직접 대입해도 '가'가 저장됨
```

유니코드 값을 16진수로 표현하여 문자를 직접 저장할 수도 있습니다:

```java
char var5 = '\u0041';  // 유니코드 16진수 표기법으로 'A' 표현
char var6 = '\uAC00';  // 유니코드 16진수 표기법으로 '가' 표현
```

`char` 타입은 정수 타입이므로 산술 연산도 가능합니다:

```java
char c1 = 'A';
char c2 = (char)(c1 + 1);  // 'B'가 저장됨 (65 + 1 = 66, 'B'의 유니코드)
System.out.println(c2);    // 출력: B
```

이러한 특성 덕분에 문자 간 비교나 정렬 등의 작업도 숫자 비교로 간단히 처리할 수 있습니다.

> 참고: `char`는 유니코드의 기본 다국어 평면(BMP, U+0000 ~ U+FFFF)까지만 표현할 수 있으며, 이모지나 일부 특수문자는 surrogate pair로 처리됩니다.

char 타입 변수에는 빈 문자<sup>empty</sup>를 대입할 수 없습니다. 빈 문자로 초기화하려고 하면 컴파일 에러가 발생하므로, 대신 공백 문자<sup>유니코드 32</sup>를 사용해 초기화해야 합니다.
```java
char c1 = ''; // 컴파일 에러
char c2 = ' '; // 공백 하나를 포함해 초기화
```

### 2.4 실수 타입 (float vs double)

자바에서 실수를 저장할 수 있는 기본 타입은 `float`와 `double`입니다. 두 타입은 저장 크기, 정밀도, 허용 범위 등에서 차이를 보입니다.

| 타입    | 메모리 크기     | 허용 범위                                | 유효 소수점 이하 자리수 | 가수 비트 수 |
|---------|------------------|-------------------------------------------|--------------------------|----------------|
| float   | 4바이트 (32비트) | ±1.4E-45 ~ ±3.4028235E+38                 | 약 7자리                 | 23비트         |
| double  | 8바이트 (64비트) | ±4.9E-324 ~ ±1.7976931348623157E+308      | 약 15~16자리             | 52비트         |

> 참고: float은 소수점 이하 약 7자리까지, double은 약 15~16자리까지 정밀하게 표현할 수 있습니다. 매우 큰 수 또는 정밀한 계산에는 double 사용이 일반적입니다.
double라는 이름은 float보다 약 2배의 정밀도를 갖는다는 의미에서 붙여졌습니다.

자바는 IEEE 754 표준에 근거해 float 타입과 double 타입의 값을 부동 소수점(floating-point) 방식으로 메모리에 저장합니다. 부동소수점이란 소수점 위치가 숫자마다 바뀜(필요에 따라 조정됨)을 의미합니다.

float 타입과 double 타입은 가수와 지수를 저장하기 위해 전체 bit를 다음과 같이 나누어 사용합니다:

| 타입 | 저장 방식 | 비트 수 |
|------|-----------|---------|
| float | 부호 1 + 지수 8 + 가수 23 | 32비트 |
| double | 부호 1 + 지수 11 + 가수 52 | 64비트 |

**예시로 보는 float의 메모리 구조**

`6.5`(10진수)를 float에 저장할 때의 비트 구조:

| 부호 (1비트) | 지수 (8비트)   | 가수 (23비트)                     |
|--------------|----------------|----------------------------------|
| 0            | 10000001       | 10100000000000000000000          |

> 메모리가 담기는 원리는 길어져서 별도의 페이지로 분류했습니다.

실수 리터럴의 기본 타입은 double입니다. 
컴파일러는 실수 리터럴을 기본적으로 double 타입으로 해석하기에, float 타입에 값을 저장하려면 리터럴 뒤에 'F' 또는 'f'를 붙여야 합니다. 

```java
double pi = 3.14159;
float pi2 = 3.14159F;  // F를 붙이지 않으면 컴파일 오류

double e = 2.71828e0;  // 지수 표기법 사용 가능 (2.71828 × 10^0)
float e2 = 2.71828e0F;
double y = 0.12E-2;    // 0.12 × 10⁻² = 0.0012
```

> 실수 연산 시 주의: 이진수로 실수를 정확히 표현할 수 없는 경우가 있어 미세한 오차가 발생할 수 있습니다. 정확한 금융 계산 등에는 BigDecimal 클래스를 사용하는 것이 좋습니다.

### 2.5 논리 타입 (boolean)

논리 타입은 `boolean` 키워드로 선언하며, 오직 `true`와 `false` 두 가지 값만 가질 수 있습니다.

```java
boolean stop = true;
boolean isActive = false;
```

boolean 타입 변수는 주로 조건문과 제어문의 실행 흐름을 제어하는 데 사용됩니다. 비교 연산이나 논리 연산의 결과값은 모두 boolean 타입입니다.

```java
// 비교 연산 결과 저장
int x = 10;
boolean result1 = (x == 20);  // false: x의 값이 20인가?
boolean result2 = (x != 20);  // true: x의 값이 20이 아닌가?
boolean result3 = (x > 0);    // true: x가 0보다 큰가?

// 논리 연산자와 함께 사용
boolean result4 = (x > 0 && x < 100);  // true: x가 0보다 크고 100보다 작은가?
boolean result5 = (x < 0 || x > 50);   // false: x가 0보다 작거나 50보다 큰가?
boolean result6 = !(x == 10);          // false: x가 10이 아닌가? (부정)
```

boolean 타입은 메모리에 내부적으로 어떻게 저장되는지에 대한 명세가 JVM에 따라 다를 수 있습니다. 실제로는 1바이트(byte) 크기의 공간을 사용하는 경우가 많지만, 자바 언어 명세에서는 정확한 크기를 명시하지 않습니다.

boolean 타입은 다음과 같은 상황에서 특히 유용합니다:

1. **상태 표현**: 프로그램의 특정 상태를 나타낼 때
   ```java
   boolean isRunning = true;
   boolean hasError = false;
   ```

2. **조건문**: 코드 실행 흐름을 제어할 때
   ```java
   if (isRunning) {
       // 실행 중일 때만 처리
   }
   
   while (!hasError) {
       // 오류가 없는 동안 반복 실행
   }
   ```

3. **메서드 반환 값**: 작업의 성공/실패 또는 특정 조건 충족 여부를 알릴 때
   ```java
   public boolean isValidUsername(String username) {
       return username != null && username.length() >= 4;
   }
   ```

논리 타입은 단순하지만 프로그램의 논리 구조를 구성하는 가장 기본적인 요소 중 하나로, 복잡한 조건식이나 상태 관리에 필수적인 역할을 합니다.

### 2.6 문자열 타입

> String은 자바 기본 타입에 속하지 않는 참조자료형<sup>Reference types</sup>입니다.

자바에서 단일 문자를 표현할 때는 작은따옴표(`'`)로 감싸 표현하지만, 큰따옴표(`"`)로 감싼 여러 개의 문자들은 유니코드로 변환되지 않습니다.

```java
char ch1 = 'A';  // 올바른 문자 선언
// 잘못된 문자 선언 예시
char ch2 = "A";  // 컴파일 에러: 큰따옴표는 String 타입에만 사용 가능
char ch3 = "홍길동";  // 컴파일 에러: 여러 문자를 char에 할당 불가
```

큰따옴표(`"`)로 감싼 문자들을 문자열이라 부르는데, 문자열을 변수에 저장하고 싶다면 다음과 같이 String 타입을 사용해야 합니다.

```java
String var1 = "A";
String var2 = "홍길동";
```

문자열 내부에 역슬래쉬(`\`)가 붙은 이스케이프<sup>escape</sup>문자를 사용할 수 있습니다. 특정 문자를 포함할 수 있고, 출력에 영향을 미칩니다.

| 이스케이프 문자 | 의미 |
|--------------|------|
| \n | 줄 바꿈 (New Line) |
| \t | 탭 (Tab) |
| \\ | 역슬래시 (Backslash) |
| \' | 작은따옴표 (Single Quote) |
| \" | 큰따옴표 (Double Quote) |
| \r | 캐리지 리턴 (Carriage Return) |
| \b | 백스페이스 (Backspace) |
| \f | 폼 피드 (Form Feed) |
| \uXXXX | 유니코드 문자 (Unicode) |

```java
System.out.println("안녕하세요.\n반갑습니다.");
// 출력:
// 안녕하세요.
// 반갑습니다.

System.out.println("이름:\t홍길동");
// 출력: 이름:    홍길동

System.out.println("\"자바\"는 재미있습니다.");
// 출력: "자바"는 재미있습니다.
```

Java 13부터 제공하는 텍스트 블록 문법을 사용하면 이스케이프나 라인피드를 할 필요 없이 작성된 그대로 문자열로 저장됩니다.

```java
String html = """
              <html>
                  <body>
                      <h1>제목</h1>
                      <p>내용</p>
                  </body>
              </html>
              """;

String query = """
               SELECT id, name, email
               FROM members
               WHERE id > 10
               ORDER BY name
               """;
```

텍스트 블록에서 줄바꿈은 `\n`에 해당합니다. 줄바꿈을 하지 않고 다음 줄에 이어서 작성하려면 맨 끝에 `\`을 붙여 주면 됩니다.<sup>Java 14</sup>

```java
String text = """
              이 문장은 \
              하나의 줄로 \
              연결됩니다.""";
// "이 문장은 하나의 줄로 연결됩니다."로 저장됨

// 공백 제어 예시
String text2 = """
               Line 1
                 Line 2
               Line 3
               """;
// 들여쓰기가 보존됨
```
